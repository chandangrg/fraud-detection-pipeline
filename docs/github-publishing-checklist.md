# GitHub publishing checklist

- Create issues for each remaining improvement; do not upload the repository as one unexplained commit.
- Use feature branches and pull requests with test evidence.
- Replace placeholder CODEOWNERS handles and badge URLs after publishing.
- Run CI before tagging `v2.0.0`.
- Record a short demo showing one happy path and one controlled failure/recovery path.
- Add screenshots only after the application is actually running.
- Never backdate commits or invent throughput, latency, uptime, or coverage numbers.
